# 커스텀 메뉴 셋 & 셋으로 슬롯 돌리기 — 설계 문서

- **작성일**: 2026-07-01
- **대상 앱**: 오늘 뭐먹지 슬롯 (`ohneul-slot/`)
- **목표**: 사용자가 이름 있는 커스텀 메뉴 셋(내장 메뉴 + 직접 입력 혼합)을 만들어, 카테고리 줄의 ⭐셋 칩으로 선택해 기존 슬롯을 돌린다.

---

## 1. 개요 & 범위

사용자가 자기만의 메뉴 묶음("회사 점심", "주말 외식")을 만들어 슬롯을 돌릴 수 있게 한다.

- **재사용**: 셋 항목이 후보 풀이 되어 기존 `pickThree` + `SlotMachine`을 그대로 사용한다(새 룰렛 UI 없음).
- **신규**: 데이터 모델(CustomSet), 저장 CRUD, 셋 칩 줄, 셋 편집 화면, 메뉴 검색.

### 결정된 사항 (확정)
| 항목 | 결정 |
|------|------|
| 돌리는 방식 | 기존 슬롯 재사용 (셋 = 후보 풀) |
| 셋 구성 | 내장 246 풀 선택 **+** 직접 입력 (둘 다) |
| 셋 개수 | 여러 개, 이름 있는 셋 |
| 진입 | 카테고리 줄에 ⭐셋 칩 추가 (탭=선택, 길게=편집), `+ 셋`=새로 만들기 |
| 관리 화면 | 셋 편집기(이름 + 항목 추가/삭제 + 저장/삭제) |

### 비범위 (YAGNI)
- 별도 룰렛 휠 UI, 셋 공유/동기화, 셋 정렬/색상 커스터마이즈, 백엔드.

## 2. 데이터 모델

`src/core/types.ts`:
```ts
export interface CustomSet {
  id: string;        // 'set:<uid>'
  name: string;
  items: Menu[];     // Menu 스냅샷 (내장=복사, 직접입력=합성)
}
```

- **내장 항목**: `MENUS`의 `Menu`를 그대로 복사 저장한다(조회 불필요, 데이터 변경에 견고).
- **직접 입력 항목**: `makeCustomMenu(name)`가 합성 `Menu`를 만든다.
  - `id`: `custom:<uid>`
  - `name`: 사용자 입력(트림)
  - `emoji`/`category`: `findCanonicalMenu(name)`로 매칭되면 그 메뉴의 값을 복사, 매칭 실패 시 `emoji: '🍽️'`, `category: 'soloeat'`(중립 기본값 — 가중치/필터에 영향 없음, 표시용)
  - 아이콘 렌더는 기존 `MenuIcon`/`resolveMenuIconUrl`가 처리(전용→정규화매칭→카테고리→이모지).

### 히스토리 자기완결화
커스텀 항목을 `먹었어요 기록`하면 `menuId`가 `custom:*`이라 `MENUS` 조회로는 이름을 복원할 수 없다. → `HistoryEntry`에 스냅샷 필드 추가:
```ts
export interface HistoryEntry {
  date: string; menuId: string; category: Exclude<CategoryId,'all'>;
  name?: string; emoji?: string;  // 추가: 표시용 스냅샷
}
```
- `addHistory`는 당첨 메뉴의 `name`/`emoji`를 함께 저장한다.
- `HistoryView`는 `entry.name`/`entry.emoji`가 있으면 그것을, 없으면(구버전) `MENUS` 조회로 폴백한다.

## 4. UI & 선택 상태 (`src/App.tsx` + 컴포넌트)

### 선택 상태 확장
- `selectedSetId: string | null` 추가.
- 카테고리 칩 선택 → `selectedSetId = null`. 셋 칩 선택 → `selectedSetId = <id>`.
- 활성 후보: `selectedSetId ? customSets.find(s=>s.id===selectedSetId)!.items : getCandidates(category)`
- 셋 삭제/소실 시 `selectedSetId`를 `null`로 되돌리고 카테고리(전체) 복귀.

### 컴포넌트
| 컴포넌트 | 책임 |
|----------|------|
| `CategoryPicker` (기존) | 내장 카테고리 칩만 (변경 없음) |
| `SetChips` (신규) | ⭐셋 칩들 + `+ 셋` 칩. onSelect / onEdit(길게) / onCreate |
| `SetEditor` (신규) | 이름 입력 · 항목 리스트(출처 뱃지·개별 삭제) · `+ 항목 추가` 시트 · 저장 · 삭제 |
| `MenuSearch` (신규) | 편집기 내부. `내장에서 찾기` 탭에서 246 검색→추가 / `직접 입력` 탭 |

- 홈 헤더/레이아웃: 카테고리 줄(`CategoryPicker`) **아래**에 `SetChips` 줄 추가.
- 편집기는 모달/풀스크린(HistoryView처럼 `position:fixed` 오버레이) 패턴 재사용.

## 5. 코어 로직 (순수 함수, Vitest 대상)

`src/core/customSet.ts` (신규):
- `makeCustomMenu(name: string): Menu` — menuMatch 활용해 합성 Menu 생성.
- `canSpinSet(set: CustomSet): boolean` — 스핀 가능 최소 **항목 ≥ 2**.

`src/core/menu.ts` (추가):
- `searchMenus(query: string): Menu[]` — `normalizeMenuName` 기반 부분일치 검색(상위 20개).

스핀: `App.spin()`이 후보만 셋 항목으로 바꾸고 `pickThree(candidates, history, favSet, today, Math.random)` 그대로 호출. 항목 2개면 2릴, 3개 이상이면 3릴(SlotMachine 기존 동작).

## 6. 엣지 & 검증

- 셋 항목 **< 2개면 "돌리기" 비활성** + 안내 문구.
- 직접 입력: 빈/공백 문자열 거부, 트림, 셋 내 **중복 이름 방지**, 항목 **상한 20**.
- 셋 이름: 빈 이름 거부(기본값 "내 셋 N").
- 셋 삭제 확인. 삭제된 셋이 선택 중이면 전체 카테고리로 복귀.
- 커스텀 당첨도 공유/즐겨찾기/기록 정상 동작(id 기반).

---

## 7. 테스트 (Vitest)

- `makeCustomMenu`: 매칭되는 이름("얼큰김치찌개"→김치찌개 emoji/category), 미매칭("존재안함xyz"→🍽️/soloeat), 트림.
- `searchMenus`: 부분일치("국밥"→돼지국밥 등 포함), 빈 쿼리 처리.
- `canSpinSet`: 0/1/2/3 항목 경계.
- `HistoryEntry` 스냅샷: addHistory가 name/emoji 저장(상태 훅은 통합 검증, 순수 변환은 위 함수로 커버).
- pickThree는 셋도 `Menu[]`라 기존 테스트로 커버.
