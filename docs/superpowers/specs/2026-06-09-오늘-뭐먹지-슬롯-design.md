## 1. 제품 개요

> **점심 메뉴 결정을 슬롯머신으로 5초에 끝내는 미니앱.**

평일 점심마다 다시 열게 만들고(daily trigger), 결과 공유로 친구를 끌어와(viral) 7월 AU를 극대화하는 것이 목표다.

### 승리 전략 (AU 최적화)
- **매일 열 이유**: 평일 점심 = 구조적 daily trigger + 점심 푸시 알림으로 재방문 유도
- **신규 유입**: 결과 공유 카드 → 받은 사람도 "나도 돌려보기"로 전환
- **이탈 방지**: 확률 가중치(같은 메뉴 반복 방지) + 주간 히스토리로 체감 유용성 ↑
- **테마 적합**: 반복되는 "뭐 먹지" 결정의 번거로움 자동화 = 테마 정중앙

### 제약 / 컨텍스트
- 솔로 빌더, AI 가속, JS/HTML5/PixiJS 베테랑, 슬롯/확률 모델링 8년 경력 → 본 앱에 구조적으로 유리
- 6/8~6/30 빌드, 7월 운영(AU 경쟁) → MVP는 확실히 완성 가능한 범위로 고정, 고급 기능은 7월 업데이트

## 3. 사용자 플로우

```
[홈] 카테고리 선택(전체 / 한식 / 중식 / 양식 / 혼밥 / 회식 …)
  → [슬롯] 릴 3개가 드르륵 회전 → 탁·탁·탁 순차 정지 + 햅틱 + 사운드
  → [결과] "오늘은 국밥!" + [공유 카드] [다시 돌리기] [먹었어요 기록]
  → [히스토리] 이번 주 캘린더(무엇을 먹었는지), 중복 방지 반영
```

- 첫 진입 마찰 0: 아무 등록 없이 바로 돌릴 수 있음(내장 메뉴 풀).
- "먹었어요 기록"을 누르면 히스토리에 저장되고, 다음 추첨의 확률 가중치에 반영된다.

## 5. 아키텍처 & 기술 스택

- **방식**: WebView 미니앱 — `create-ait-app`
- **프레임워크**: React + Vite (토스 예제 `weekly-todo-react` 표준 참고)
- **슬롯 연출**: Canvas 2D 또는 CSS transform (가벼움 우선). PixiJS는 선택(과할 수 있음)
- **상태 관리**: Zustand (또는 React Context + reducer)
- **플랫폼 API**: `@apps-in-toss/framework` — 저장소 / 공유 / 햅틱 / 푸시
- **백엔드**: 없음 (전부 클라이언트 완결). 푸시 스케줄은 플랫폼 기능 활용
- **설정**: `granite.config.ts` — `appName`, `displayName`, `primaryColor`, `icon`을 콘솔 등록값과 일치

## 6. 모듈 경계 (각 단위 = 단일 책임, 인터페이스로만 소통)

| 모듈 | 책임 | 인터페이스(개념) | 의존 |
|------|------|-----------------|------|
| `menu/` | 내장 메뉴 풀 데이터 + 카테고리 필터 | `getCandidates(category) → Menu[]` | 없음(순수) |
| `picker/` | 확률 가중치 추첨 엔진 | `pick(candidates, history) → Menu` | 없음(순수) |
| `slot/` | 슬롯 릴 애니메이션 + 햅틱/사운드 | `play(result) → Promise<void>` | 햅틱 API |
| `history/` | 먹은 기록 저장·조회·주간 집계 | `add(entry)`, `getWeek()`, `getRecent(days)` | 저장소 API |
| `favorites/` | 단골 후보 저장·조회 | `add/remove/list()` | 저장소 API |
| `share/` | 결과 공유 카드 생성·공유 | `shareResult(menu)` | 공유 API |
| `notify/` | 점심 푸시 등록/해제 | `enable(time)`, `disable()` | 푸시 API |

- 플랫폼 API 호출은 각 래퍼 모듈 내부에 격리 → 로컬에서 목(mock)으로 대체 가능, 테스트·교체 용이.
- `picker`, `menu`, `history`의 집계 로직은 순수 함수 → 단위 테스트 대상.

## 7. 확률 가중치 로직 (차별화 포인트)

각 메뉴의 추첨 가중치:

```
weight = base × recencyDecay(menu, history) × favoriteBoost(menu)
```

- `recencyDecay`: 최근에 먹은 메뉴일수록 감소
  - 예: 어제 먹음 ×0.1 · 3일 전 ×0.5 · 7일 이상 ×1.0 (계수는 튜닝 대상)
- `favoriteBoost`: 즐겨찾기 메뉴는 소폭 가중(예: ×1.3)
- 가중 랜덤 추첨으로 당첨 결정 → "또 같은 거 나오네" 불만 제거 = 체감 품질 ↑
- **연출은 랜덤처럼 보이되 결과는 가중** — 슬롯 RTP 튜닝 감각을 그대로 적용
